# 障碍物生成约束算法

### 虚拟网格系统

- 使用虚拟网格系统规范障碍物位置，所有障碍物只能放置在格子的正中心位置。
- 每个格子大小为1x1米，网格宽度 = Road Width - Road Margin × 2

## 一、避免漏洞和特殊情况（基础）

### 1. 所有障碍物不得重叠
**每个网格最多只能出现一种障碍物，障碍物所在格子的周围8个格子不得出现障碍物。**
- 使用 `Dictionary<float, HashSet<int>> occupiedGrids` 记录每行已占用的网格坐标
- 使用周围8个格子检查来确保障碍物之间有足够空间

### 2. 同个障碍物不得连续大量出现，避免难度过高或过低
**在任意10个连续出现的障碍物中，同个障碍物出现次数不得超过4次**
- 使用 `List<ObstacleType> recentObstacleHistory` 记录最近10个障碍物类型

### 3. 避免游戏漏洞，防止玩家贴墙走避开大部分伤害
**在任意10个连续出现的障碍物中，至少3个障碍物必须出现在最边缘的格子**
- 使用 `List<bool> recentEdgeObstacles` 记录最近10个障碍物是否在边缘

## 二、基于得分的动态难度系统（重点）

引入玩家当前得分"x"作为变量，得分"x"越高游戏难度越高。

具体表现为"x"越高，障碍物越多越密集，一些比较难对付的障碍物刚开始不出现或很少出现，随着"x"变大出现频率显著变高。障碍物难度等级在Obstacles.md已标出。

**实现方式：**

1. 根据玩家当前得分"x"调整障碍物生成参数：
   - 随着得分增加，每行生成的障碍物数量增加
   - 障碍物之间的间距减少，使得障碍物更加密集
   - 高难度障碍物的出现概率随得分增加而提高

2. 具体实现算法：
   - 定义基础分数阈值（baseScoreThreshold），表示游戏难度开始明显提升的分数点
   - 使用公式计算难度系数：difficultyFactor = playerScore / baseScoreThreshold
   - 根据难度系数动态调整以下参数：
     - minObstaclesPerRow：随难度增加而增加（上限为初始maxObstaclesPerRow的2倍）
     - maxObstaclesPerRow：随难度增加而增加（上限为初始值的2倍）
     - minObstacleDistance：随难度增加而减小（下限为初始值的50%）
     - maxObstacleDistance：随难度增加而减小（下限为初始值的60%）

3. 障碍物类型选择：
   - 通过权重系统控制不同难度等级障碍物的出现率
   - 设置每种障碍物的初始权重和最大权重
   - 随着难度系数增加，高难度障碍物权重逐步提高
   - 每次生成障碍物时，基于当前各障碍物权重随机选择

4. 性能优化：
   - 使用插值算法实现难度平滑过渡，避免游戏难度突变
   - 对权重计算结果进行缓存，减少每帧重复计算
   - 在玩家得分变化显著时才重新计算难度参数

## 三、符合常识的关系机制

### 1. 火炬树桩在豌豆射手、双重射手、三重射手前面的出现率应该更高
**实现方式：**
- 检测后方一定范围内是否有豌豆射手类型的障碍物（豌豆射手、双重射手、三重射手）
- 若检测到，则提高火炬树桩的权重（增加其出现概率）
- 实现自适应调整：距离射手越近，火炬树桩的出现概率越高
- 使用 `Dictionary<float, Dictionary<int, ObstacleType>> rowObstacleTypes` 检查后方行的障碍物类型
