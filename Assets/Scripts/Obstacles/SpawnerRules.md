# 障碍物生成约束算法

### 虚拟网格系统

- 使用虚拟网格系统规范障碍物位置，所有障碍物只能放置在格子的正中心位置。
- 每个格子大小为1x1米，网格宽度 = Road Width - Road Margin × 2

## 一、避免漏洞和特殊情况（基础）

### 1. 所有障碍物不得重叠
**每个网格最多只能出现一种障碍物，障碍物所在格子的周围8个格子不得出现障碍物。**
- 使用 `Dictionary<float, HashSet<int>> occupiedGrids` 记录每行已占用的网格坐标
- 使用周围8个格子检查来确保障碍物之间有足够空间

### 2. 同个障碍物不得连续大量出现，避免难度过高或过低
**在任意10个连续出现的障碍物中，同个障碍物出现次数不得超过4次**
- 使用 `List<ObstacleType> recentObstacleHistory` 记录最近10个障碍物类型

### 3. 避免游戏漏洞，防止玩家贴墙走避开大部分伤害
**在任意10个连续出现的障碍物中，至少3个障碍物必须出现在最边缘的格子**
- 使用 `List<bool> recentEdgeObstacles` 记录最近10个障碍物是否在边缘

## 二、基于得分的动态难度系统

游戏使用玩家跑出的距离（米）作为得分，难度随得分动态调整。
随着玩家得分变高，障碍物变多变密，高难度障碍物出现率变高。

### 难度等级分类
- **等级1（简单）**：豌豆射手、寒冰射手、坚果、土豆地雷
- **等级2（中等）**：双重射手、高坚果、火焰树桩、冰菇、仙人掌、火爆辣椒
- **等级3（困难）**：三重射手、猫尾草、樱桃炸弹

### 实现方式
- **平滑更新**：每 5 米刷新一次，过渡平滑。
- **难度系数**：
  - 0–200m：1.0 → 1.15
  - 200–400m：1.15 → 1.3
  - 400–800m：2.0 → 2.3
  - 800–1600m：2.3 → 2.5
  - 1600–3200m：2.5 → 3.2
  - 3200–6400m：3.2 → 3.6
  - 6400–12800m：3.6 → 4.5
  - 12800m+：5.0（常量）
- **统一驱动规则**：
  - 数量：min×[1.00→1.20]，max×[1.00→1.20]
  - 间距：初始×[2.00→1.00]
  - 类型权重：≥200m 开放等级2，≥400m 开放等级3；难度越高，等级1↓、等级2↑、等级3↑。

## 三、符合常识的关系机制

### 1. 火炬树桩在豌豆射手、双重射手、三重射手前面的出现率应该更高
**实现方式：**
- 检测后方一定范围内是否有豌豆射手类型的障碍物（豌豆射手、双重射手、三重射手）
- 若检测到，则提高火炬树桩的权重（增加其出现概率）
- 实现自适应调整：距离射手越近，火炬树桩的出现概率越高
- 使用 `Dictionary<float, Dictionary<int, ObstacleType>> rowObstacleTypes` 检查后方行的障碍物类型
